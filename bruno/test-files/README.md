# Bruno test fixtures

Binary fixtures used by `.bru` requests. These are NOT generated by Claude — add them locally.

## Existing fixtures

- `test-image.png` — small valid PNG. Used by:
  - `Vendix/Organization/Upload/*.bru`
  - `Vendix/Store/Checkout/Checkout - Multipart With Receipt.bru`

## Missing fixtures (you must add)

- `test-bad.exe` — any binary whose MIME falls OUTSIDE the checkout receipt allowlist
  (`image/jpeg`, `image/png`, `image/webp`, `application/pdf`).
  Used by `Vendix/Store/Checkout/Checkout - Multipart Bad MIME.bru` to validate the
  `VALIDATION_FILE_TYPE` (400) response.

  Quick creation (any of these works):

  ```bash
  # Option A: a minimal fake PE header
  printf 'MZ\x90\x00\x00\x00\x00\x00' > bruno/test-files/test-bad.exe

  # Option B: copy any random non-allowed binary
  cp /bin/ls bruno/test-files/test-bad.exe
  ```

  Size should stay well under 5 MB so the request is rejected by MIME, not by size.
